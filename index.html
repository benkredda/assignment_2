Node_a Module

My first ROS Node

Overview

This module implements a ROS node that serves as an action client, allowing the user to set a target (x, y) or cancel it. The node subscribes to the /odom topic for odometry data and publishes the robot’s position and velocity as a custom message on the position_and_velocity topic.

scripts.Node_a.target()[source]

Function for setting a target position.

This function allows the user to set the positions (x, y) of the target. It creates a goal of type PoseStamped and sends it to the action server using the send_goal function, which is a member function of the SimpleActionClient class. The goal is sent to the action server using the /reaching_goal topic and the PlanningAction action type. The robot will move to reach the target.

Node_b Module

My second ROS Node

Overview

This node is responsible for printing the number of goals reached and canceled. It subscribes to the /reaching_goal/result topic to receive data and calls the callback_subscriber function when a message of type PlanningAction/Result is received.

Upon receiving a message on the /reaching_goal/result topic, the callback_subscriber function is triggered. It increments the global variables canceled_goals and reached_goals based on the status message received, where the status can be represented as an integer. Additionally, a service is created and called to print the number of reached and canceled goals in the terminal. To send the request and trigger the printing, you need to run the following command in a separate terminal:

$ rosservice call /reach_cancel_goal

This command invokes the service and triggers the printing of the number of reached and canceled goals by Node_b.

Usage

To use this node, follow these steps:

  1. Launch the necessary ROS nodes and topics required for communication.

  2. Run the Node_b Python script to start the service node.

  3. In a separate terminal, use the rosservice call command as shown above to send the request and print the goals information.

Node_c Module

My third ROS Node

Overview

This node subscribes to the robot’s position and velocity using the custom message odom_custom_msg. It listens to the /odom_custom topic for incoming messages of type odom_custom_msg. Upon receiving a message, it calculates and prints the distance of the robot from the target and the robot’s average velocity.

Parameters can be used to configure the publishing frequency of the information. The print_dist parameter sets the frequency at which Node_c publishes the information.

Additionally, a launch file is provided to start the simulation, which includes the necessary nodes and configurations. The launch file sets the value for the publishing frequency of Node_c. You can adjust this value to control how frequently the information is published.

To visualize the simulation and the relationships between topics, nodes, etc., you can generate the computational graph. Execute the following command in the terminal:

$ rqt_graph

This command will display the computational graph.

scripts.Node_c.callback_subscriber(data)[source]

Callback function for Node C.

This function is called when a message is received on the topic /odom_custom, where the type of the message is odom_custom_msg. The function computes the distance of the robot from the target and the average velocity of the robot based on the last 5 messages, and then prints the results.

Parameters

data (odom_custom_msg) – The received message containing robot’s position and velocity information.
