Overview
---------
This node subscribes to the robot's position and velocity using the custom message `odom_custom_msg`. It listens to the `/odom_custom` topic for incoming messages of type `odom_custom_msg`. Upon receiving a message, it calculates and prints the distance of the robot from the target and the robot's average velocity.

Parameters can be used to configure the publishing frequency of the information. The `print_dist` parameter sets the frequency at which Node_c publishes the information.

Additionally, a launch file is provided to start the simulation, which includes the necessary nodes and configurations. The launch file sets the value for the publishing frequency of Node_c. You can adjust this value to control how frequently the information is published.

To visualize the simulation and the relationships between topics, nodes, etc., you can generate the computational graph. Execute the following command in the terminal:

   $ rqt_graph

This command will display the computational graph.

---------

[docs]def callback_subscriber(data):
    """
    Callback function for Node C.

    This function is called when a message is received on the topic `/odom_custom`,
    where the type of the message is odom_custom_msg. The function computes the distance of the robot
    from the target and the average velocity of the robot based on the last 5 messages, and then prints the results.

    Args:
        data (odom_custom_msg): The received message containing robot's position and velocity information.
    """
    
    # Function code :
    global counter
    global temp_vel
    global avg_vel
    global des_pos_distance


    # Get desired position from the parameter server
    
    des_pos_x = rospy.get_param("/des_pos_x")
    des_pos_y = rospy.get_param("/des_pos_y")

    # Get desired position from the parameter server
    cur_pos_x = data.x
    cur_pos_y = data.y

    # Compute the distance of the robot from the target

    des_pos_distance= math.sqrt(((des_pos_x - cur_pos_x)**2)+((des_pos_y - cur_pos_y)**2))


    # Get current velocity of the robot

    cur_vel_x = data.vel_x
    cur_vel_y = data.vel_y

    # Compute the current velocity of the robot
    
    cur_vel= math.sqrt(((cur_vel_x)**2)+((cur_vel_y)**2))


    
    if counter<5:
        # Compute the average velocity of the robot using the last 5 messages

        temp_vel=temp_vel+cur_vel
        counter +=1

    elif counter==5:

        counter=0
        temp_vel /= 5
        avg_vel=temp_vel
        temp_vel=0

if __name__ == "__main__":

    start_description(start_description_flag)

    # Create a rate object using the parameter /print_dist to control the publishing rate of the node

    HZ=rospy.get_param("/print_dist")
    rate = rospy.Rate(HZ)

    # Subscribe to the topic /position_and_velocity where the message type is odom_custom_msg

    rospy.Subscriber("position_and_velocity", odom_custom_msg, callback_subscriber)
    # Node C runs until it is shutdown it prints the sequence number, the data print_dist ,it prints the distance of the robot from the target and the average velocity of the robot ,then sleeps for the time set in the rate object
    
    while not rospy.is_shutdown():
        print(f"Sequence: {sequence}")
        print(f"Data print distance: {HZ} HZ")
        print(f"Distance to target: {des_pos_distance:.3f}")
        print(f"Robot’s average velocity: {avg_vel:.3f}")
        print("---------------------------")
        sequence += 1
        rate.sleep()
